=== Cyber Security - CIA Triad Resume Materi ===
[27 Mar 2026]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 APA ITU CYBER SECURITY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Definisi : Melindungi sistem, jaringan, dan aplikasi dari serangan
Fokus    : Melindungi DATA DIGITAL dari 3 ancaman utama →
           → Data dilihat pihak tidak berwenang (Data Seen)
           → Data diubah tanpa izin (Data Changed)
           → Data tidak sampai/tidak bisa diakses (Data Not Delivered)
Penting  : Dulu info disimpan di kertas, sekarang digital & online
           → tanpa keamanan = bisa bocor, dimanipulasi, atau hilang
Wajib    : Kebutuhan inti bagi pemerintah, organisasi, dan individu

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 CIA TRIAD - TIGA PILAR UTAMA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CIA bukan hanya definisi → ini adalah SECURITY MINDSET profesional
Setiap insiden keamanan selalu dikaitkan ke salah satu dari 3 pilar ini

          🔒 Confidentiality
                 ▲
                / \
               /   \
    ✅ Integrity ── ✅ Availability

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 1. CONFIDENTIALITY (Kerahasiaan)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Definisi : Data sensitif HANYA bisa diakses oleh individu BERWENANG
Jika gagal→ kerugian finansial, pelanggaran privasi, konsekuensi hukum

Analogi nyata  : Ngobrol privat → orang asing diam-diam dengarkan
                 → solusi : ngobrol di tempat aman, waspada sekitar
Analogi digital: Login medsos di WiFi publik kedai kopi
                 → kredensial dicegat → akun di-logout paksa
                 → solusi : ENKRIPSI + ACCESS CONTROL

Contoh kasus :
  ✗ No  → Kredensial Gmail ditulis di sticky note di meja kantor
  ✓ Yes → Dokumen internal hanya bisa diakses karyawan yang butuh
  ✗ No  → Dokumen pribadi tersedia bebas di internet

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 2. INTEGRITY (Integritas)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Definisi : Data TIDAK BOLEH dimodifikasi oleh pihak tidak berwenang
Jika gagal→ data tidak bisa dipercaya → bisa berakibat BERBAHAYA

Analogi nyata  : Nilai ujian diubah orang lain sebelum diserahkan
                 → solusi : catat di lembar terpisah & verifikasi dulu
Analogi digital: Transfer bank dicegat → nomor rekening tujuan diubah
                 → uang masuk ke tempat yang salah
                 → solusi : berbagai teknik integrity (dipelajari nanti)

Contoh kasus :
  ✓ Yes → Data diubah melalui persetujuan yang berwenang
  ✗ No  → Catatan kehadiran siswa diubah setelah dikunci guru
  ✗ No  → Harga pesanan dimodifikasi sebelum checkout

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 3. AVAILABILITY (Ketersediaan)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Definisi : Data & layanan HARUS TERSEDIA untuk pengguna berwenang
           saat mereka membutuhkannya
Fakta    : Downtime singkat saja = kerugian besar bagi bisnis
           Bisnis modern = sangat bergantung pada layanan digital

Analogi nyata  : Uang di bank aman, tapi bank tutup karena mati listrik
                 → uang ada tapi tidak bisa diakses = tidak berguna
                 → solusi : generator listrik cadangan
Analogi digital: Penyerang kirim ribuan request sekaligus → website DOWN
                 (ini disebut serangan DDoS)
                 → tidak ada data bocor/diubah, tapi tetap rugi besar
                 → solusi : kelola beban traffic, blokir request berlebih

Contoh kasus :
  ✗ No  → Layanan kritis terganggu karena instalasi software
  ✗ No  → Website perusahaan offline saat jam kerja
  ✓ Yes → Semua sistem bisa diakses karyawan selama jam kerja

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 SECURITY MINDSET - CARA BERPIKIR PROFESIONAL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Saat insiden terjadi, selalu tanya 3 pertanyaan ini →

  1. Apakah data sensitif terekspos ke pihak tidak berwenang?
     → Confidentiality terdampak

  2. Apakah data dimodifikasi tanpa izin?
     → Integrity terdampak

  3. Apakah sistem/layanan tidak tersedia saat dibutuhkan?
     → Availability terdampak

Tujuan : Nilai dampak insiden → tentukan respons yang tepat

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 ISTILAH PENTING YANG WAJIB HAPAL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
CIA Triad        = Confidentiality, Integrity, Availability
                   → Fondasi dari seluruh konsep cyber security
Confidentiality  = Informasi digital tidak tersedia ke pihak tidak berwenang
Integrity        = Informasi digital tidak dimodifikasi tanpa izin
Availability     = Informasi digital tidak menjadi tidak tersedia saat dibutuhkan
Enkripsi         = Teknik mengacak data agar tidak bisa dibaca pihak lain
Access Control   = Mekanisme pembatasan siapa yang boleh akses data
DDoS             = Serangan banjir request → membuat layanan tidak tersedia
Security Mindset = Pola pikir keamanan berbasis CIA Triad
                   (bukan hanya hafal definisi, tapi bisa analisis insiden)
Authorized       = Resmi / sudah mendapat izin dari pemilik sistem
